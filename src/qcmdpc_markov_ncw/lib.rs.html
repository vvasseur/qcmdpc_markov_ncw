1//! # QC-MDPC Markovian Model
2//!
3//! This program implements a Markovian model to calculate the Decoding Failure
4//! Rate (DFR) of QC-MDPC codes using the step-by-step algorithm.
5//! We consider a QC-MDPC code `(h0, h1) ‚àà (ùîΩ‚ÇÇ[x](x ≥ ‚àí 1))¬≤` where `|h0| = |h1|
6//! = d` for which we know that there exists near codewords:
7//! * `(x^i * h0, 0)` for `i` in `{0, ..., r}`,
8//! * `(0, x^i * h1)` for `i` in `{0, ..., r}`.
9//!
10//! These are of type `(d, d)`, producing error vectors of weight `d` with
11//! syndrome weight `d`. Near codewords of type `(a, b)` with small `a` and `b`
12//! are known to hinder decoder performance, causing the error floor phenomenon.
13//!
14//! This model accounts for these near codewords by using states `(s, t, u)`
15//! where:
16//! * `s` is the syndrome weight
17//! * `t` is the error weight
18//! * `u` is the number of common bits in the nearest near codeword
19//!
20//! By doing this, the model can predict an error floor.
21//!
22//! ## References
23//!
24//! > \[ALMPRTV25]: Sarah Arpin, Jun Bo Lau, Antoine Mesnard, Ray Perlner, Angela Robinson, Jean-Pierre Tillich & Valentin Vasseur: Error floor prediction with Markov models for QC-MDPC codes. <https://eprint.iacr.org/2025/153>
25//!
26//! > \[SV19]: Nicolas Sendrier & Valentin Vasseur: On the Decoding Failure Rate
27//! > of QC-MDPC Bit-Flipping Decoders. <https://doi.org/10.1007/978-3-030-25510-7_22>
28//!
29//! ## Models
30//!
31//! ### Specific keys (¬ß6 of \[ALMPRTV25])
32//!
33//! You can specify either a custom degree distribution or a code file using
34//! `--degrees` or `--code`. The `--degrees` option expects a JSON string
35//! representing the distribution of the degrees of the check nodes in the
36//! subgraph generated by the near codeword (for each block).
37//!
38//! While you can provide either option, internally the program always converts
39//! code files to degree distributions (this conversion happens automatically
40//! with `--code`). You can also explicitly convert a code file to its degree
41//! distribution using the `compute-degrees` command and then use that output
42//! with `--degrees`.
43//!
44//! **Format**
45//!
46//! The `--degrees` option expect a JSON array containing two arrays (one for
47//! each block), where each array contains objects with "degree" and "count"
48//! fields representing the degree distribution. For example:
49//! ```json
50//! [[{"degree":0,"count":1577},{"degree":1,"count":17},{"degree":2,"count":122},{"degree":4,"count":7}],
51//!  [{"degree":0,"count":1576},{"degree":1,"count":16},{"degree":2,"count":126},{"degree":4,"count":4},{"degree":5,"count":1}]]
52//! ```
53//!
54//! The `--code` option expects a file containing the code description (one
55//! element of the support per line, blocks separated by an empty line).
56//!
57//! #### Default: Perfect keys (Appendix D of \[ALMPRTV25])
58//!
59//! By default, this model considers "perfect" keys. These keys have distance
60//! spectra for each block `h0`, `h1` with multiplicity at most 1. This implies
61//! that the support of the blocks are simple cyclic difference sets.
62//!
63//! If you need to actually construct such a code, for example if you want to
64//! confirm the results of the model by comparing it with simulation data, these
65//! keys can be constructed using the Singer construction, yielding keys with `r
66//! = q^2 + q + 1` and `d ‚â§ q + 1` for any prime power `q`.
67//!
68//! ### ST Model (\[SV19])
69//!
70//! You can use the simpler, earlier, model (ST model) by specifying the `--st`
71//! option along with a xi parameter. This model uses states `(s, t)` instead of
72//! `(s, t, u)`.
73//!
74//! ### Threshold Function
75//!
76//! The threshold function can be specified using the `--threshold` option as a
77//! list of mathematical expressions. These expressions can use variables `s`
78//! (syndrome weight) and `d` (column weight) and standard mathematical
79//! functions. Non-integer values are automatically rounded by default.
80//!
81//! **Examples**
82//! * Simple majority threshold: `--threshold '["(d+1)/2"]'`
83//! * Floor of linear function: `--threshold '["floor(0.006016213884791455 * s +
84//!   8.797325112097532)"]'`
85//! * Ceiling of linear function: `--threshold '["ceil(0.006016213884791455 * s
86//!   + 8.797325112097532)"]'`
87//! * Multiple independent thresholds: `--threshold '["0.006016213884791455 * s
88//!   + 10.797325112097532", "0.006016213884791455 * s + 8.797325112097532",
89//!   "(d+1)/2"]'`
90//!
91//! ## Subcommands
92//!
93//! 1. `transitions`: Calculate absorbing probabilities for each state in the
94//!    Markov chain.
95//!
96//!    Computes the probability of eventually reaching each absorbing state
97//! (Success, Blocked, NearCodeword)    from every possible decoder state. This
98//! is equivalent to computing the limit of A^n as n‚Üí‚àû, where A
99//!    is the one-step transition matrix.
100//!
101//!    **Usage**
102//!    ```text
103//!    transitions \
104//!        --r <block_length> \
105//!        --d <block_weight> \
106//!        --pass <t_pass> \
107//!        --fail <t_fail> \
108//!        [--output <output_file>] \
109//!        [--degrees <custom_degree_distribution> | --code <code_file>] \
110//!        [--threshold <threshold_function>] \
111//!        [--st <xi_factor>]
112//!    ```
113//!    Output: zstd compressed binary file containing absorbing probabilities
114//!    for each state.
115//!
116//! 2. `initial-states`: Calculate initial state distribution for a given error
117//!    weight.
118//!
119//!    **Usage**
120//!    ```text
121//!    initial-states \
122//!        --r <block_length> \
123//!        --d <block_weight> \
124//!        --t <error_weight> \
125//!        [--output <output_file>] \
126//!        [--degrees <custom_degree_distribution> | --code <code_file>] \
127//!        [--st <xi_factor>]
128//!    ```
129//!    Output: zstd compressed binary file containing initial state
130//! probabilities.
131//!
132//! 3. `dfr`: Combine initial state distribution with absorbing probabilities to
133//!    compute the final DFR.
134//!
135//!    **Usage**
136//!    ```text
137//!    dfr \
138//!        --transitions <transitions_file> \
139//!        --initial-states <initial_states_file> \
140//!        [--output <output_file>] \
141//!        [--st <xi_factor>]
142//!    ```
143//! Output: JSON file containing detailed DFR analysis:
144//! - For ST model: `{"dfr_blocked": 0.001234}`
145//! - For STUB model: `{"dfr_ncw": 0.001, "dfr_blocked": 0.002, "by_u": [...]}`
146//!   where `by_u` contains detailed breakdown by u parameter values with
147//!   conditional and absolute probabilities for each u value.
148//!
149//! 4. `compute-degrees`: Compute the degree distribution of near codewords in
150//!    the Tanner graph.
151//!
152//!    Usage:
153//!    ```text
154//!    compute-degrees \
155//!        --r <block_length> \
156//!        --code <code_file>
157//!    ```
158//! Output: JSON string containing the degree distribution
159//!
160//! ## Example Workflow
161//!
162//! A typical session using this model involves the following steps:
163//! (Using `r = 1723` as an example)
164//!
165//! 1. Compute absorbing probabilities: ```sh cargo run --release -- transitions
166//!    --r 1723 --d 17 --pass 1 --fail 200 --code code.txt --output
167//!    transitions.json.zstd ```
168//!
169//! 2. Compute initial state distribution: ```sh cargo run --release --
170//!    initial-states --r 1723 --d 17 --t 55 --code code.txt --output
171//!    initial_states.json.zstd ```
172//!
173//! 3. Compute final DFR: ```sh cargo run --release -- dfr --transitions
174//!    transitions.json.zstd --initial-states initial_states.json.zstd --output
175//!    final_dfr.json ```

177/// Alpha parameter used in distribution trimming
178///
179/// We keep the `(1 - 10^-ALPHA)` most frequent values in intermediate
180/// distributions. Higher values mean more precision but slower computation.
